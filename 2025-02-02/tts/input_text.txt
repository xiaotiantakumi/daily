# 「ガバナンスって何から始めるの？」

オフィスの窓から差し込む柔らかな日差しの中で、美咲はパソコンを覗き込みながら小声でつぶやいた。  
「ガバナンス設計って、どこから手を付けたらいいのか全然わからないなぁ……。」

横に座っていた田中 悠がにっこりと笑って問いかけた。「ガバナンスについて何か特定のことが気になってるの？」

「うーん、Azure でリソースをどう管理するのか、具体的にイメージが湧かなくて……。」

悠は優しく頷いた。「まずは、Azure におけるガバナンスの基本から見ていこう。ガバナンスは組織環境の制御を維持するためのメカニズムとプロセスを提供してくれるんだ。つまり、どういう風にリソースを管理し、どんなルールを設定するべきかを決めるのがガバナンスなんだよ。」

## 「ガバナンスの基本構造」

「ガバナンス戦略を適用するにはまず組織の環境を階層構造で整理する必要があるんだ。」悠は続けた。

「階層構造って何ですか？」美咲が首を傾げた。

「Azure の階層構造は大まかに 4 つのレベルに分かれているんだ。管理グループ、サブスクリプション、リソースグループ、そしてリソースね。」

「管理グループは、複数のサブスクリプションをまとめるためのもので、ポリシーやコンプライアンスの管理に便利だよ。」悠がにっこりと補足する。

「なるほど、じゃあサブスクリプションはどういう役割なんですか？」

悠はさらに説明を加えた。「サブスクリプションは管理とスケールの単位として機能する論理コンテナだよ。課金の単位にもなるから、プロジェクトやチームごとに異なる課金を管理するのに役立つんだ。」

## 「リソースの管理」

「次はリソースグループとリソースだね。」悠は画面を示しながら続けた。「リソースグループは Azure リソースがデプロイ・管理される単位で、。」

「リソースって具体的に何ですか？」美咲が尋ねた。

「リソースは Azure 上で作成されるサービスのインスタンスのことだ。例えば、仮想マシンやストレージ、SQL データベースなどがそれに当たるよ。」

「なるほど！それぞれの階層が役割を持っているんですね。」美咲は感心したように言った。

## 【まとめ】Azure ガバナンスのポイント

- ガバナンスはリソースの管理と、組織の最適な制御を維持するための戦略を決めること。
- 組織環境を階層構造（管理グループ、サブスクリプション、リソースグループ、リソース）で整理することで、ガバナンスを適切に適用できる。
- 管理グループは複数のサブスクリプションをまとめてポリシーを適用するのに役立つ。
- サブスクリプションは管理とスケールの論理単位で、課金の単位にもなる。
- リソースグループは Azure リソースのデプロイ・管理単位。
- リソースは Azure で作成されるサービスのインスタンス。

「これでガバナンスの最初のステップは分かったかな？」

「はい、とてもスッキリしました！」美咲は明るい表情で感謝を伝えた。
